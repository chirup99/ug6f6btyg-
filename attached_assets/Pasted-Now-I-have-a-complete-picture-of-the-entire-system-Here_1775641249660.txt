LAYER 3 — The Real Ticker (Candle Builder + SSE Broadcaster)
File: server/angel-one-real-ticker.ts
This is the heart of the system. One singleton AngelOneRealTicker manages all connected chart clients.
addClient() — called when a chart opens
addClient(clientId, res, symbol, symbolToken, exchange, tradingSymbol, initialOhlc, intervalSeconds) {
  // 1. Set SSE response headers (keeps HTTP connection open)
  res.writeHead(200, { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', ... });
  // 2. Calculate current candle's start time
  const now = Math.floor(Date.now() / 1000);
  const candleStartTime = Math.floor(now / intervalSeconds) * intervalSeconds;
  // e.g. if now=43200 (12:00 PM) and interval=900 (15min), candleStartTime = 43200
  // 3. Store client state including seeded candle OHLC
  this.clients.set(clientId, {
    id: clientId, res, symbol, symbolToken,
    candleOhlc: { open, high, low, close, candleStartTime, intervalSeconds },
    ...
  });
  // 4. Subscribe to Angel One WebSocket for live ticks
  this.subscribeToWebSocket(client);
  // 5. Start the 700ms fallback broadcast if not already running
  if (!this.broadcastInterval) this.startBroadcast();
}
subscribeToWebSocket() — the tick callback
const tickCallback = (wsData) => {
  const ltp = wsData.close; // LTP comes as .close from WebSocket layer
  // CHECK market hours first
  if (!this.isMarketOpen(client.exchange)) {
    // Just send the last known price, marked as 'closed'
    client.res.write(`data: ${JSON.stringify({ ...closedPrice, marketStatus: 'closed' })}\n\n`);
    return;
  }
  // Calculate if time boundary crossed (new candle?)
  const currentCandleStart = Math.floor(currentTime / intervalSeconds) * intervalSeconds;
  const isNewCandle = currentCandleStart > client.candleOhlc.candleStartTime;
  if (isNewCandle) {
    // Reset OHLC with LTP as new Open
    client.candleOhlc = { open: ltp, high: ltp, low: ltp, close: ltp, candleStartTime: currentCandleStart, ... };
  } else {
    // Update H/L/C within same candle
    client.candleOhlc.high  = Math.max(client.candleOhlc.high, ltp);
    client.candleOhlc.low   = Math.min(client.candleOhlc.low,  ltp);
    client.candleOhlc.close = ltp;
  }
  // Push updated candle over SSE immediately (not waiting for 700ms)
  client.res.write(`data: ${JSON.stringify(livePrice)}\n\n`);
};
Key insight: Ticks from WebSocket are pushed immediately to the client (no 700ms wait). The 700ms interval is only a fallback for when no tick has arrived yet.

this.broadcastInterval = setInterval(() => {

Why 700ms Specifically?
The 700ms is defined in two places:
1.	Fallback broadcast loop (angel-one-real-ticker.ts line 340): setInterval(..., 700) — sends last known data if no tick arrived
2.	Market data poll on frontend (advanced-candlestick-chart.tsx line 416): refetchInterval: 700 — polls /api/market-data for live price indicator
Real ticks from Angel One's WebSocket arrive faster than 700ms (can be sub-second) and are pushed immediately — the 700ms is a safety heartbeat, not the actual update frequency.

Summary of What You See in the Screenshot (the count timer 3:07)
That 3:07 label is the candle countdown timer — time remaining in the current candle interval. It's calculated from:
timeRemaining = intervalSeconds - (currentTime - candleStartTime)
Where candleStartTime comes from client.candleOhlc.candleStartTime on the server, sent in each SSE message. The frontend converts that into MM:SS format and displays it on the top-right of the candle.
